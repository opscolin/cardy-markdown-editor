# Cardy Markdown Editor - Project Prompts

本项目是一个基于 React 的精美 Markdown 卡片编辑器。以下是整理后的完整开发提示词，可用于引导 AI 助手进行功能迭代或项目重建。

---

## 核心需求描述

创建一个名为 **Cardy** 的 Web 应用，旨在将 Markdown 内容转换为 3:4 比例的精致笔记卡片。应用应具备“左侧编辑、右侧实时预览”的双栏布局，并支持多卡片分页和多种导出格式。

### 1. 技术栈要求
- **框架**: React 18+ (使用 TypeScript)
- **构建工具**: Vite
- **样式**: Tailwind CSS (现代、极简风格)
- **图标**: Lucide React
- **动画**: Motion (framer-motion)
- **核心库**: 
  - `react-markdown` (解析 MD)
  - `html-to-image` (生成图片)
  - `jspdf` (生成 PDF)
  - `jszip` (打包 ZIP)

### 2. 核心功能点

#### 编辑器与预览
- **双栏布局**: 左侧为无边框编辑器，右侧为卡片预览区。
- **分页逻辑**: 识别 Markdown 中的 `---` 分隔符，将内容切分为多张独立的 3:4 比例卡片。
- **图片支持**: 
  - 支持从本地选择图片或直接粘贴图片。
  - **关键实现**: 插入时将图片转换为 **Base64** 编码，以确保数据在本地存储中持久化。
  - 编辑器内应实时显示图片缩略图。
- **代码块**: 实现 Mac 风格的代码块（带红、黄、绿三色圆点装饰），支持行内代码高亮。

#### 样式与定制
- **等比缩放**: 提供 XS/SM/BASE/LG 四档字号。
  - **实现逻辑**: 卡片内部所有尺寸（字体、间距、边距）使用 `em` 单位，通过调整根容器的 `font-size` 实现整体等比缩放。
- **背景网格**: 可一键切换卡片的背景网格线（Grid Background）。
- **自定义页脚**: 每份笔记可设置独立的页脚文本（如：@Username 或 版权信息）。

#### 数据持久化
- **自动保存**: 使用 `LocalStorage` 实时保存所有笔记列表、当前激活的笔记 ID 以及用户的偏好设置。
- **状态反馈**: 在标题栏显示 "Saved" 呼吸灯，告知用户保存状态。

#### 导出功能
- **单张导出**: 每张卡片悬浮显示导出按钮，支持导出为 PNG。
- **批量导出**: 
  - **PDF**: 将所有卡片按顺序合并为一个高质量 PDF 文件。
  - **ZIP**: 将所有卡片分别生成图片并打包为压缩包。
- **导出优化**: 导出时需禁用 `cacheBust` 以兼容 Base64/Blob 链接。

### 3. UI/UX 设计规范
- **配色**: 纯白背景，灰色边框 (`gray-200`)，极简主义。
- **侧边栏**: 可折叠，用于管理笔记列表（新建、切换、删除）。
- **标题栏**: 包含笔记标题编辑、保存状态、导出按钮、设置项（字号、网格、页脚）以及 GitHub 链接。
- **交互**: 
  - 导出按钮在执行时应显示 Loading 状态。
  - 按钮状态需细化，避免多个导出按钮同时显示加载动画。

### 4. 关键避坑指南 (Tips)
- `react-markdown` 默认过滤 `blob:` 协议，需配置 `urlTransform`。
- 使用 `contentEditable` 实现编辑器以支持图片内联展示，并编写同步函数将 HTML 转回 Markdown 源码。
- 确保卡片容器具有 `shrink-0` 和 `aspect-[3/4]` 属性，防止布局塌陷。
